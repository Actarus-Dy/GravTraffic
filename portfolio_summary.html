GravTraffic

Et si on simulait les embouteillages avec les mêmes équations que celles qui gouvernent les galaxies ?

L'idée en 30 secondes

Dans l'espace, les étoiles s'attirent par la gravité. Les étoiles massives attirent tout vers elles — exactement comme une voiture lente sur l'autoroute qui « aspire » les voitures derrière elle dans un bouchon.


J'ai pris cette analogie au pied de la lettre : dans GravTraffic, chaque voiture est traitée comme un corps gravitationnel. Une voiture lente a une masse positive — elle attire les autres, elle crée un bouchon. Une voiture rapide a une masse négative — elle repousse les autres, elle fluidifie le trafic.


Le résultat ? Les embouteillages, les ondes de choc, les convois — tout émerge naturellement des équations de la gravité, sans qu'on programme la moindre règle de conduite.

L'analogie gravité ↔ trafic

🌑
Trou noir = Bouchon
Un objet massif attire tout vers lui.
Un véhicule lent « aspire » les voitures derrière lui.
💨
Énergie noire = Fluidité
L'énergie noire repousse et accélère l'expansion.
Un véhicule rapide repousse et disperse les voisins.
🌊
Onde gravitationnelle = Onde de choc
Quand deux étoiles fusionnent, une onde se propage.
Quand une voiture freine, le ralentissement remonte la file.
🌀
Galaxie = Convoi
Les étoiles s'organisent en structures par gravité.
Les voitures forment spontanément des pelotons.

Comment ça marche

On mesure l'écart de vitesse

Chaque voiture est comparée à la vitesse moyenne du trafic. Plus lente = masse positive (attracteur). Plus rapide = masse négative (repousseur).

On calcule les forces gravitationnelles

Comme Newton l'a fait pour les planètes : chaque voiture exerce une force sur toutes les autres. Même formule, adaptée au trafic.

On fait avancer le temps

Un intégrateur « leapfrog » (le même que dans les simulations de galaxies) met à jour les positions et vitesses de manière stable.

On observe ce qui émerge

Sans aucune règle de conduite programmée, les bouchons se forment, les ondes de choc remontent la file, le trafic s'auto-organise.

Ce que j'ai construit

⚙️
Moteur de physique
Calcul N-corps gravitationnel avec algorithme Barnes-Hut (le même qu'en astrophysique), optimisé pour des milliers de véhicules.
🚀
Accélération ×323
Compilation JIT avec Numba : le code Python est transformé en code machine natif, 323 fois plus rapide que le Python standard.
🌐
API REST temps réel
Serveur FastAPI avec WebSocket pour streamer la simulation en direct, prédiction T+15 minutes, métriques Prometheus.
📊
Dashboard interactif
Visualisation en temps réel dans le navigateur : carte des véhicules, heatmap de congestion, contrôles play/pause, 5 scénarios.
🐳
Déploiement Docker
Dockerfile multi-stage, docker-compose dev/prod, pipeline CI/CD GitHub Actions automatisé (lint → tests → build).
🔬
Validation scientifique
Le modèle reproduit le diagramme fondamental du trafic (R² = 0.98) et génère des phénomènes d'émergence mesurables.

Ce que j'ai appris

Ce projet m'a permis de toucher à beaucoup de domaines : physique computationnelle (simulation N-corps, intégrateurs symplectiques), algorithmique avancée (Barnes-Hut, quadtree), optimisation haute performance (Numba JIT, GPU), développement web (API REST, WebSocket, dashboard), DevOps (Docker, CI/CD, monitoring Prometheus), et rigueur scientifique (validation statistique, revues systématiques par un agent Devil's Advocate).


Le point le plus fascinant : il suffit d'appliquer les équations de la gravité pour que les embouteillages apparaissent tout seuls. Pas besoin de programmer « si la voiture devant freine, je freine ». La physique s'en charge.
